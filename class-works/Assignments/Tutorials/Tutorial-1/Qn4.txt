In C++  enums are user defined data types used to assign mnemonic names to integer constants. They make code vastly more readable, maintainable, and less error by replacing "magic numbers" (like 0, 1, 2) with descriptive words. for ex:

#include <iostream>
#include <string>
using namespace std;

enum direction {
    east,
    west,
    north,
    south
};

// Array order must perfectly match the enum order
const string directionNames[] = {"east", "west", "north", "south"};

int main() {
    for (int a = 0; a < 4; a++) {
        cout << directionNames[a] << endl;
    }
    return 0;
}


An identifier is a unique name assigned to program elements such as variables, functions, arrays, classes, or objects. The rule for defining are:
1. You can only use uppercase letters (A-Z), lowercase letters (a-z), digits (0-9), and underscores (_).
2. C++ is strictly case-sensitive. The compiler treats TotalSalary, totalsalary, and TOTALSALARY as three completely separate variables.
3. You cannot use words that are reserved by the C++ language (such as int, double, class, return, or if) as identifier names.
4. Whitespaces, punctuation marks, and special symbols (like @, $, #, %, -) are completely forbidden.






When projects grow large, or when we use multiple libraries, we face the problem of name collision. This happens when two different parts of a program try to use the exact same name for a variable, class, or function. A namespace acts like a a folder that groups related code together. It lets us use identical function names in the same program without causing a conflict. for ex:
#include <iostream>
using namespace std;

namespace MyGame {
    void render() {
        cout << "Rendering the game player character..." << endl;
    }
}

namespace AdvancedGraphics {
    void render() {
        cout << "Rendering a complex 3D background grid..." << endl;
    }
}

int main() {
    MyGame::render();           
    AdvancedGraphics::render(); 

    return 0;
}
